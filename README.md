# Економічний Онлайн-Симулятор "Місто" (MVP)

Це кросплатформений економічний симулятор та соціальна стратегія (гібрид Mafia + Cashflow + Monopoly). Проект розробляється для запуску на мобільних пристроях (iOS/Android) та ПК.

## 🎯 Мета гри
Головна мета "Міста" — дати гравцеві відчуття соціального підйому всередині живої економіки. Гравець починає як звичайний громадянин із мінімальним капіталом, кімнатою в хостелі, обмеженою енергією та першими простими можливостями. Далі він поступово будує власний шлях: знаходить роботу, платить за житло, навчається, отримує кращі посади, відкриває бізнеси, впливає на міську економіку та може дійти до політичної або бізнесової влади.

Гра має навчати фінансової грамотності через наслідки виборів, а не через лекції. Кожне рішення повинно мати зрозумілий економічний або соціальний ефект: робота дає гроші, але забирає енергію; освіта відкриває нові можливості, але потребує інвестицій; швидкі тіньові рішення можуть прискорити прогрес, але створюють ризик штрафів, втрати репутації чи втручання поліції.

## 🔁 Перша ігрова петля MVP
Перший пріоритет розробки — зробити коротку, приємну та зрозумілу петлю на 5 хвилин гри:

1. Гравець реєструється як новий громадянин.
2. Отримує стартовий баланс і кімнату в хостелі.
3. Переглядає доступні вакансії та влаштовується на роботу.
4. Відпрацьовує зміну, отримує зарплату, платить податок і витрачає енергію.
5. Платить за сон/оренду, відновлює енергію та настрій.
6. Накопичує на освіту, складає іспит і відкриває кращу роботу.
7. Бачить, як його дії впливають на власний статус і показники міста.

Поки ця петля не відчувається як гра, додаткові системи на кшталт спорту, картелів, страхування, профспілок і політики варто розглядати як майбутні "двері", а не як головну складність MVP.

## 🧠 Мотиваційна психологія
Основний мотиватор гри — не щоденні бонуси заради бонусів, а відчуття особистого прогресу. На старті гравець має обрати або отримати першу життєву ціль: вибратися з хостелу, стати менеджером, відкрити перший бізнес, стати відомим спортсменом, вплинути на податки або балотуватися в мери.

Після кожної дії гра повинна показувати не лише зміну чисел, а й наближення до цілі. Наприклад: "Ви заробили 180 ₴ і стали на 12% ближче до сертифіката менеджера". Це створює відчуття незавершеної історії, яку хочеться продовжити.

Базові шкали прогресу:

* **Стабільність:** житло, енергія, настрій, борги, регулярний дохід.
* **Репутація:** чесна освіта, історія роботи, довіра, відсутність поліцейських записів.
* **Вплив:** бізнеси, спортивні клуби, участь у міській політиці, здатність змінювати правила економіки.

Ці шкали мають допомогти гравцеві бачити не просто баланс грошей, а власну позицію в соціальній системі міста.

## 🚀 Стек технологій MVP
* **Клієнт (Client):** [Godot Engine 4.x](https://godotengine.org/) (мова програмування: C#).
* **Бекенд (Backend):** [Python](https://www.python.org/) + FastAPI + SQLAlchemy.
* **База даних:** PostgreSQL одразу з першого MVP.
* **Realtime/session target:** Redis пізніше, коли перша ігрова петля стане стабільною.

## 🧰 Що треба встановити

Обов'язково:

1. **Python 3.12+** для backend.
2. **Godot 4.3 .NET edition** для клієнта.
3. **.NET SDK**, сумісний із версією Godot .NET.
4. **PostgreSQL 16+** або **Docker Desktop** для запуску Postgres контейнером.
5. **Git** для контролю версій.
6. **PowerShell 7** для локальних dev-скриптів.

Корисно, але не обов'язково:

* VS Code або Rider.
* pgAdmin або DBeaver для перегляду Postgres.
* Bruno, Insomnia або Postman для ручної перевірки API.
* Redis - пізніше, коли локальна петля стане стабільною.

Godot MCP addon уже входить у проект і використовується для змін сцен через AI. Інші Godot addons поки не ставимо: dashboard, jobs, sleep і education мають лишатися простою стабільною MVP-петлею.

## 🐘 PostgreSQL

Варіант A: через Docker Desktop:

```powershell
docker compose up -d postgres
```

Варіант B: встановити PostgreSQL 16+ напряму і створити бази:

```sql
CREATE USER city WITH PASSWORD 'city_dev_password';
CREATE DATABASE city_game OWNER city;
CREATE DATABASE city_game_test OWNER city;
```

Проєкт очікує змінні з `.env.example`:

```text
CITY_DATABASE_URL=postgresql+psycopg2://city:city_dev_password@127.0.0.1:5432/city_game
CITY_TEST_DATABASE_URL=postgresql+psycopg2://city:city_dev_password@127.0.0.1:5432/city_game_test
```

## ⚙️ Локальний запуск backend

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r backend\requirements-dev.txt
python -m alembic upgrade head
python -m uvicorn backend.main:app --reload
```

Налаштування середовища дивись у `.env.example`.

## 🧪 Тести

```powershell
python -m pytest
```

Перший набір тестів перевіряє seed міста і базову петлю `work -> sleep`.
Тести інтеграційні та потребують `CITY_TEST_DATABASE_URL`.

Smoke-перевірка повного HTTP MVP loop:

```powershell
python scripts\smoke_mvp.py
```

Локальна перевірка перед комітом:

```powershell
.\scripts\check.ps1
```

Якщо системний terminal відкрив старий PowerShell, можна явно запускати через portable PowerShell 7:

```powershell
.\scripts\pwsh7.ps1 -NoProfile -File .\scripts\check.ps1
```

Playtest із чистої dev-бази та повною перевіркою перед запуском Godot:

```powershell
.\scripts\play.ps1 -ResetDb -RunCheck
```

Локальний Godot MCP bridge для роботи зі сценами через відкритий Godot editor:

```powershell
.\scripts\start_godot_mcp_bridge.ps1
.\scripts\invoke_godot_mcp.ps1 -Tool get_errors -ArgsJson '{"include_warnings":true}'
```

Cursor читає `.cursor/mcp.json` і запускає локальний Python MCP server `scripts\godot_mcp_stdio.py`. Якщо Cursor не бачить Godot tools, перезапустіть Cursor після запуску Godot editor і bridge.

Повторні POST-запити для `work`, `sleep` та `exam` можна захистити header-ом:

```http
Idempotency-Key: any-unique-action-key
```

Якщо той самий ключ повториться для тієї самої дії, backend поверне попередній результат без повторного нарахування або списання.

Мутації конкретного гравця також потребують token, який повертається після реєстрації:

```http
X-Player-Token: player-auth-token
```

Godot-клієнт зберігає цей token у локальній сесії та додає його до `apply`, `work`, `sleep`, `exam` і запиту статусу гравця.

У репозиторії також є GitHub Actions workflow `.github/workflows/ci.yml`: на `push` і `pull_request` він запускає PostgreSQL 16, Alembic migrations, backend pytest і Godot C# build.

## 🧱 Міграції БД

```powershell
python -m alembic upgrade head
python -m alembic revision --autogenerate -m "describe change"
```

Під час dev-start backend також викликає `upgrade head`, щоб локальна база не відставала від моделей.

## 🕒 Dev day tick

У debug-режимі доступний ручний перехід на наступний ігровий день:

```http
POST /api/city/tick-day
```

Він застосовує добову оренду, легке зниження енергії/настрою та перерахунок інфляції. Це MVP/dev-інструмент, не фінальний production scheduler.

## 📂 Цільова структура проекту
```directory
city_economic_simulator\
├── client\                  # Клієнтська частина гри (Godot Engine 4 C#)
│   ├── assets\              # Графічні ресурси, звуки, шрифти
│   ├── scenes\              # Сцени інтерфейсу (City, Hostel, Jobs, Mafia, Monopoly)
│   ├── scripts\             # C# Скрипти логіки та UI
│   └── project.godot        # Файл конфігурації проекту Godot
├── backend\                 # Серверна частина гри (FastAPI / Python)
│   ├── app\
│   │   ├── api\             # Ендпоінти для авторизації, ринку праці, ЖКГ
│   │   ├── core\            # Налаштування, безпека, підключення до БД
│   │   ├── models\          # Моделі бази даних (SQLAlchemy/SQLModel)
│   │   ├── schemas\         # Валідація даних (Pydantic)
│   │   ├── services\        # Економічна логіка, симуляція Скарбниці
│   │   └── websockets\      # Реалтайм-сервер для румів та чату
│   ├── requirements.txt     # Python залежності
│   └── main.py              # Точка входу в сервер
└── database\                # Конфігурації БД та міграції
    └── schema.sql           # Базова схема бази даних
```

Поточний технічний план розвитку винесений у `ROADMAP.md`.

## 📌 Наступні кроки для запуску розробки
1. **Налаштування Робочої Області:** Відкрийте папку `C:\ces` у VS Code, Godot або іншому редакторі.
2. **База Даних:** Створення первинної схеми бази даних (таблиці гравців, вакансій, фірм, логів транзакцій).
3. **Бекенд:** Ініціалізація FastAPI проекту та налаштування WebSocket підключення.
4. **Клієнт:** Створення базового проекту Godot 4 (C#) та налаштування перших Glassmorphic сцен для дашборду міста.
